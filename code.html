Code & reproducibility

The general-construct pipeline, in run order. Every figure, table, and bundle on this site is regenerable from the census + embeddings with these scripts.

1 · Census generation

  • RUBRIC_general_v1.md — the general-complementarity rubric (consumption + production)
  • regenerate_census_general.py — GPT-4o primary census, both economies, two-pass + fuzzy resolution (its gpt-4o-mini pass is legacy and unused in the paper)
  • regenerate_census_gemini.py — cross-vendor Gemini-2.5-Pro replication census (the paper's replication generator)

2 · Consumer-final labeling

  • classify_consumer_final.py — GPT-4o + mini consumer-final classifier
  • classify_consumer_final_gemini.py — Gemini classifier (cross-vendor consensus)

3 · Embedding analytics

  • general_analyze.py — position spectrum, comp-vs-sub cosine AUC, by-type
  • general_rank_only.py / general_rank_probe.py — percentile/retrieval + linear probe
  • general_probe_only.py — probe AUC
  • The economy×domain 2×2 is computed from the cross-vendor consumption/production census (xvendor labels) and rendered by figpub_general.py (fig_2x2).

4 · Graph (companion-exclusive)

  • graph_general.py — complement graph on the general census: Louvain communities + modularity, hubs, reciprocity, edges (both economies)

5 · Figures

  • figpub_general.py, figpub_general_r2.py — spectrum, distributions, percentile, probe, product/service, 2×2
  • figpub_general_dose.py — dose-response (Figure 14)
  • make_umap_figures.py — UMAP maps + focal/ego figures

6 · Audits (reliability & coverage)

  • dimension_agreement.py — cross-generator reliability of the type tag and the four complementarity-dimension tags on the shared pairs (Cohen's κ, prevalences; deterministic)
  • coverage_audit.py — audit of focal categories without complements, by NAPCS section, both economies (Appendix A, Table A.1)

Environment: Python 3, numpy / pandas / scikit-learn / pyarrow; umap-learn for the maps; networkx + python-louvain for the graph; matplotlib for figures. API keys (OpenAI, Gemini) are required only to regenerate the census/labels, not the analytics.<
