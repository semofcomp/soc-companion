Methods, in plain language

How the census is built, labeled, and measured — the moving parts behind every figure and table.

1 · How complements are generated

For each NAPCS category (the focal), a large language model proposes the product/service categories genuinely used together with it — serving distinct, mutually reinforcing roles — under a general rubric that spans both consumption (a household's joint use) and production (inputs/equipment/services used together in a process). A two-pass pipeline first free-generates candidate phrases, resolves them to real NAPCS categories by fuzzy matching, then re-selects and ranks the six best, tagging each as product/service and with relationship dimensions (functional, sequential, occasion, demand-spillover). Substitutes and near-duplicates of the focal are excluded by construction.

2 · How consumption vs. production is labeled

Every category is classified consumer-final vs not by an LLM judge (acquired by households for final use, vs intermediate/capital/wholesale/business), run under two vendors (GPT-4o and Gemini-2.5-Pro). The website's domain label is the cross-vendor consensus: consumption = both agree consumer-final, production = both agree non-consumer (a positively-labeled class), disagreements marked review. Cross-vendor agreement is high (Canada 93%, US 87%). A complement-neighborhood "audit" relabeling was considered and deliberately not used — it would entangle the split with the very complement graph it partitions, and complementarity legitimately crosses the consumption/production line.

3 · Anchors

Two reference sets calibrate every measurement: substitutes — taxonomic siblings (same 6-digit code in Canada, same trilateral code in the US) — and random category pairs. Intermediate taxonomic levels (same group / same class, coarser prefixes) provide the "similarity gradient." Complements are read against these same anchors so a position, percentile, or AUC always carries the same meaning.

4 · The four representations

Each category's text is embedded four ways: TF-IDF (lexical overlap), GloVe (word co-occurrence), and the Gemini and OpenAI contextual transformers. The dense embeddings are anisotropy-corrected (mean-centered over the universe, then L2-normalized) before any cosine, so each model's cosines are meaningful against that model's own random and substitute baselines. Cross-model comparisons never use raw cosine levels — they use anchored positions, ranks, and probes (the paper's Section 6.2 shows why positions cannot be compared across models).

5 · The probe (why it recovers what cosine hides)

Plain cosine collapses the embedding to a single similarity axis, on which complements land below substitutes.
